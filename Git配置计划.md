## 🔧 配置步骤

### 阶段1：安装Git客户端
**选项A：使用包管理器安装**（推荐）
```powershell
# 使用Chocolatey（如果已安装）
choco install git -y

# 或使用Winget（Windows 10+）
winget install Git.Git
```

**选项B：手动下载安装**
1. 下载Git for Windows：https://git-scm.com/download/win
2. 运行安装程序
3. 选择默认配置

**选项C：便携版Git**
1. 下载便携版
2. 解压到指定目录
3. 添加到PATH环境变量

### 阶段2：验证Git安装
```powershell
# 验证安装
git --version

# 验证配置
git config --list
```

### 阶段3：配置Git身份
```bash
# 设置用户名和邮箱（使用你已登录的账号）
git config --global user.name "陳志標"
git config --global user.email "你的邮箱@example.com"

# 设置默认编辑器
git config --global core.editor "code --wait"  # 或 "notepad"

# 设置行尾转换
git config --global core.autocrlf true
```

### 阶段4：验证Git访问权限
```bash
# 测试与远程仓库的连接
git ls-remote https://github.com/用户名/仓库名

# 或克隆测试仓库
git clone https://github.com/用户名/测试仓库.git
```

### 阶段5：配置认证方式
**选项A：使用已保存的凭据**（如果已登录）
- Windows凭据管理器可能已保存
- 测试是否可以直接使用

**选项B：配置SSH密钥**（最安全）
```bash
# 生成SSH密钥
ssh-keygen -t ed25519 -C "xiaoling.assistant@gmail.com"

# 将公钥添加到Git平台
# 复制 ~/.ssh/id_ed25519.pub 内容
# 添加到Git平台的SSH密钥设置
```

**选项C：使用个人访问令牌**（方便）
1. 在Git平台生成访问令牌
2. 设置适当的权限
3. 使用令牌进行认证

## 🖥️ Git平台特定配置

### GitHub配置
```bash
# 设置GitHub特定配置
git config --global github.user "你的GitHub用户名"
git config --global credential.helper manager
```

### GitLab配置
```bash
# 设置GitLab特定配置
git config --global gitlab.user "你的GitLab用户名"
```

### Bitbucket配置
```bash
# 设置Bitbucket特定配置
git config --global bitbucket.user "你的Bitbucket用户名"
```

## 🔐 安全配置

### 认证安全
1. **使用SSH密钥**（推荐）
   - 生成新的专用密钥对
   - 设置密钥密码
   - 定期更换密钥

2. **访问令牌管理**
   - 设置有限权限
   - 设置过期时间
   - 定期更换令牌

3. **凭据存储**
   - 使用安全的凭据管理器
   - 不存储明文密码
   - 定期清理凭据

### 操作安全
1. **操作日志**：记录所有Git操作
2. **变更确认**：重要变更前确认
3. **备份机制**：重要仓库定期备份
4. **权限审核**：定期检查访问权限
