# 文件夹命名规范

## 原则
- **用户可见文件夹**：使用中文命名，方便用户识别和管理
- **系统内部文件夹**：使用英文命名，便于程序处理
- **项目文件夹**：根据项目性质决定，用户参与的项目用中文

## 中文文件夹示例（用户可见）

### 1. 文档类
- `文档` - 存放各种文档文件
- `笔记` - 个人笔记和记录
- `报告` - 项目报告和工作汇报
- `合同` - 合同和协议文件

### 2. 媒体类
- `图片` - 图片和照片
- `视频` - 视频文件
- `音频` - 音频和音乐文件
- `设计稿` - 设计文件和素材

### 3. 项目类
- `工作项目` - 工作相关项目
- `个人项目` - 个人兴趣项目
- `学习资料` - 学习材料和教程
- `临时项目` - 临时性项目文件

### 4. 工具类
- `工具脚本` - 各种工具脚本
- `配置文件` - 系统配置文件
- `备份文件` - 重要文件备份
- `下载文件` - 临时下载文件

## 英文文件夹示例（系统内部）

### 1. 程序类
- `scripts` - 脚本文件
- `config` - 配置文件
- `logs` - 日志文件
- `temp` - 临时文件

### 2. 开发类
- `src` - 源代码
- `dist` - 发布文件
- `node_modules` - Node.js模块
- `venv` - Python虚拟环境

### 3. 数据类
- `data` - 数据文件
- `database` - 数据库文件
- `cache` - 缓存文件
- `uploads` - 上传文件

## 混合使用示例

### 项目结构示例
```
项目名称/
├── 文档/          # 用户可见，中文
│   ├── 需求文档.md
│   ├── 设计稿/
│   └── 报告/
├── src/           # 系统内部，英文
│   ├── main.py
│   └── utils/
├── config/        # 系统内部，英文
│   └── settings.json
└── 测试数据/      # 用户可见，中文
    └── 测试用例.xlsx
```

## 注意事项

1. **一致性**：同一类别的文件夹保持命名一致
2. **简洁性**：文件夹名称要简洁明了
3. **避免特殊字符**：不要使用 `/ \ : * ? " < > |` 等特殊字符
4. **大小写敏感**：Windows不区分大小写，但Linux/macOS区分
