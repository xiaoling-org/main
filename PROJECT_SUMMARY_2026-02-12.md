## 5. 🚀 新Mac到手后无缝迁移流程

### 步骤1：环境复原
```bash
# 1. 在新Mac上安装ClawdBot
npm install -g openclaw-cn

# 2. 停止当前服务（如果有）
openclaw-cn gateway stop

# 3. 用备份文件覆盖新机配置
# 将备份的 config.yaml, .env, storage/ 等复制到新机对应位置
```

### 步骤2：权限授予（必须且关键）
在 **macOS 系统设置 > 隐私与安全性** 中，为ClawdBot进程手动开启：

1. **辅助功能** - 允许控制电脑
2. **完全磁盘访问** - 允许读写文件
3. **自动化** - 允许自动化操作

**重要提示**：此步骤每台新设备都需手动重做，权限无法迁移。

## 13. 🆘 故障排除指南

### 常见问题及解决方案：

#### 问题1：Telegram机器人无响应
```bash
# 检查服务状态
openclaw-cn gateway status

# 重启服务
openclaw-cn gateway restart

# 检查Token配置
cat config.yaml | grep telegram
```

#### 问题2：API密钥失效
1. 检查DeepSeek API Key是否有效
2. 检查Telegram Bot Token是否被撤销
3. 通过 @BotFather 重新生成Token
4. 更新配置文件并重启服务

#### 问题3：macOS权限问题
1. 打开"系统设置 > 隐私与安全性"
2. 为ClawdBot开启：辅助功能、完全磁盘访问、自动化
3. 重启ClawdBot服务
4. 如果仍不行，重启Mac

#### 问题4：数据迁移失败
1. 确认备份文件完整
2. 检查文件权限
3. 对比新旧配置文件差异
4. 逐步迁移，分步验证
